Agentic Engineering 不是 Vibe Coding。写代码变便宜了，写好代码并没有。

DEFINE THE PROBLEM RUN THE CODE VERIFY THE RESULT
Boundary / 核心边界

专业工程师用 Agent 放大能力，而不是跳过理解与审查

Vibe Coding

常用于非程序员原型；代码未审查、偏原型级；人几乎不参与代码理解。

Agentic Engineering

服务专业工程；代码经过审查、测试并可上线；人负责定义问题、验证结果、持续改进 harness。

Agent 的定义：在循环中调用工具以达成目标。Coding agent 的关键差异，是它能执行代码并迭代到“确实能跑”。

01 能跑，并且被证明能跑

02 解决对的问题

03 处理错误路径

04 简洁、可维护

05 有测试与合适文档

06 留出演进空间

07 满足安全与可观测性要求

New Constraint

代码的成本结构已经改变

旧习惯建立在“代码昂贵”之上；Agent 让实现、重构、测试与文档可以并行推进。

质量标准
不能随成本下降。
Testing / 三层防线

不是生成代码，而是证明结果

01TDD：先写测试，确认失败，再实现至通过。

02全量测试：新会话先建立测试意识，发现回归。

03手动验证：用真实命令与浏览器留下运行证据。

Human Work Moves Up

人的工作没有消失，而是上移

决定写什么代码，以及接受哪些权衡

提供工具与足够细的规格

验证结果是否稳健可信

把经验写回指令与 harness

Five Principles

五大原则层

01

定义边界：能写能跑，但必须由人定目标、给工具、验结果。

02

接受新约束：代码近乎免费，好代码仍然昂贵。

03

囤积解法：积累带可运行证明的代码与例子。

04

提高质量：让后台 agent 处理简单但耗时的清理。

05

严守反模式：绝不提交自己没审过的 PR。
